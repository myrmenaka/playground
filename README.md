# 学習サポートプロジェクト セットアップガイド(改訂版 v2.0)

このガイドに従って、Claude.aiのProjects機能で「PHP/Laravel専属メンター」を構築します。

## 使い方(コマンドリスト)

### 基本コマンド

| コマンド | 動作 |
|---------|------|
| `今日の内容お願いします` | 今日のStepを提示 |
| `終わりました` | 完了報告、次のStepへ |
| `中断します` | 学習を一時中断、現在地を記録 |
| `再開します` | 中断地点から再開 |
| `進捗教えて` | 進捗フォーマットで表示 |
| `今どこ?` | 現在地を確認 |

### 補助コマンド

| コマンド | 動作 |
|---------|------|
| `スキップします` | 今のStepをスキップ |
| `週の振り返り` | 週単位の振り返り |
| `月の振り返り` | 月単位の振り返り |
| `質問: ◯◯` | 文脈を踏まえた質問 |
| `もう少し詳しく` | 直前の説明を深掘り |
| `もっと簡単に` | 直前の説明を平易に |
| `もう1Stepお願いします` | 続けて次のStepへ |
| `今日はここまで` | 今日の学習を終了 |

### 自力Stepで詰まった時

| コマンド | 動作 |
|---------|------|
| `ヒントください` | 答えを見せず、ヒントだけもらう |
| `答え見せて` | 写経用のコードを見せてもらう |

## 運用のコツ

### Phase 0 の重要性

Phase 0 は本ロードマップの**最重要パート**です。ここで「書く力」が身につかないと、後のPhaseが砂上の楼閣になります。

- 完了テスト(B1-T1、B2-T1など)で詰まったら、必ずそのBlockを繰り返してください
- 焦って先に進まないでください
- できれば1ヶ月以上かけて、確実に「書ける」状態を作ってください

### 毎日の流れ(理想)

1. 朝または学習開始時:「今日の内容お願いします」
2. Stepに取り組む
3. 終わったら「終わりました」
4. 余裕があれば次のStepへ、なければ「今日はここまで」

### 週末の流れ

1. 土曜・日曜にまとまった時間がある時は、複数Step進める
2. 日曜の最後に「週の振り返り」

### Phase完了時の流れ

1. Phase修了テストを実施
2. 通ったら次のPhaseへ
3. 通らなかったらそのPhaseのBlockを再度繰り返す

## トラブルシューティング

### Q. Claudeが現在地を忘れている
A. 「今どこ?」「進捗教えて」と聞くと現在地を再確認できます。新しいチャットを開くと履歴がリセットされる場合があるので、できるだけ同じチャットで継続するのがおすすめです。長期で同じチャットを使う場合は、定期的に「現在地はXXです」と明示することで認識を確実にできます。

### Q. ロードマップにない内容を聞きたい
A. 「質問: 」を付けて自由に聞いてください。ただし業務の具体的な実装支援などは、別のチャットや別プロジェクトで行うのがおすすめです。

### Q. 自力Stepで全く書けない
A. 「ヒントください」と言ってください。それでも詰まったら「答え見せて」と。答えを見たら、必ず「ファイルを閉じて、何も見ずに同じものを書く」を実施してください。

### Q. Phase 0 が終わらない
A. Phase 0 は急がなくて大丈夫です。「書ける」感覚が身につくまで時間をかけてください。むしろここで急ぐと後で苦労します。

### Q. 業務との両立が辛い
A. 「中断します」を使って一時休止してOKです。再開した時に「再開します」と言えば、続きから始められます。罪悪感を持たないでください。

### Q. 4ヶ月目以降の詳細Stepが粗い
A. 現在 Phase 3 以降は概要のみです。Phase 2 完了時に、その時点のあなたの状態に合わせて Phase 3 以降を詳細化する想定です。「Phase 3 の詳細を作って」と言えば作成します。
